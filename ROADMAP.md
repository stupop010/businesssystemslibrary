# Business Systems Library - Build Roadmap

## Principle

Do not jump into writing articles.

Build the business properly from the ground up. Each stage must be completed
before moving to the next.

Primary audience: service-based businesses with 5-50 employees that want to
improve operations, systems, and efficiency.

Positioning statement: Business Systems Library helps service businesses with
5-50 employees build better systems, improve operations, and grow through
practical processes, SOPs, workflows, checklists, and AI automation.

## Phase 1 - Foundation

Goal: Create a professional brand.

Tasks:

- Decide final domain
- Secure all social media handles
- Purchase hosting
- Configure domain
- Configure email
- Create Git repository
- Decide technology stack
- Set up local development environment
- Create project documentation
- Create brand guidelines

Deliverables:

- Live domain
- Git repository
- Development environment
- Brand guide

## Phase 2 - Brand Identity

Goal: Build a brand that looks trustworthy.

Tasks:

- Logo
- Colour palette
- Typography
- Icons
- Design system
- UI components
- Illustration style
- Favicon
- Open Graph images

Deliverable: complete design system.

## Phase 3 - Information Architecture

This is probably the most important phase.

Define:

- Main categories
- Article relationships
- Navigation model
- Internal linking model
- Search model
- Tag model
- Browse paths

Deliverables:

- Master sitemap
- Taxonomy
- Category hierarchy
- URL structure
- Navigation
- Tag system
- Cross-link strategy

## Phase 4 - Content Standards

Every article should follow the same structure.

Define:

- Writing style
- Reading level
- Formatting
- Images
- Diagrams
- Flowcharts
- References
- Checklists
- Templates
- AI sections
- Automation sections
- Related articles
- Version history

Deliverable: content style guide.

## Phase 5 - Research System

Build a repeatable research workflow.

Create:

- Source evaluation criteria
- Research templates
- Citation process
- Knowledge database
- Reading workflow
- Book notes workflow
- Podcast note workflow
- Research repository

Deliverable: research operating system.

## Phase 6 - Technical Build

Develop:

- Homepage
- Category pages
- Article templates
- Search
- Filtering
- Navigation
- Author pages
- Resource pages
- Newsletter
- SEO framework
- Analytics
- Performance optimisation

Deliverable: version 1 website.

## Phase 7 - Initial Content

Do not publish random articles.

For the first 6-12 months, start with a complete model business: a plumbing
company with approximately 15 employees.

Map every process:

- Lead generation
- Lead capture
- Sales
- Quoting
- Scheduling
- Job delivery
- Invoicing
- Payment collection
- Customer service
- Customer retention

This creates a connected knowledge graph instead of isolated articles.

## Phase 8 - Expansion

Expand into:

- Construction
- Landscaping
- Cleaning
- Roofing
- Property maintenance
- Electrical
- HVAC
- IT support
- Marketing agencies
- Accountancy firms
- Recruitment agencies
- Law firms
- Professional services

Each industry should use the same framework.

## Phase 9 - Premium Resources

Eventually add:

- SOP templates
- Downloadable checklists
- Business templates
- Process maps
- AI prompts
- Automation blueprints
- Workflow downloads

## Phase 10 - AI Layer

Eventually build:

- AI assistant
- Business process recommender
- Workflow builder
- Interactive process designer
- Knowledge search
- Business diagnostics
- Automation recommendations

## Quality Rules

Never publish for the sake of publishing.

Every page must be:

- Better than competitors
- Better organised
- Better researched
- Better designed
- More practical
- More evergreen

Every article should improve the overall library.
